# Example Invocations

Use these prompts as starting points for the lead skill or its specialist agents.

## Requested Core Examples

- `Use $ux-ui-design-lead to review this settings page and simplify it around the primary user job.`
- `Use $ux-ui-design-lead in icon/label design mode to design a Remix button that makes sense without relying on the icon alone, then recommend tooltip, label, and accessibility additions.`
- `Use $ux-ui-design-lead to compare this running UI against these screenshots and close the gap in hierarchy, spacing, and semantics.`
- `Use $ux-ui-design-lead to redesign this dashboard around the two most common user jobs.`
- `Use $ux-ui-design-lead to review this mobile screen for hierarchy, scanability, and thumb ergonomics.`

## Additional Useful Examples

- `Use $ux-ui-design-lead in critique mode on this onboarding flow from a first-time user perspective.`
- `Use $ux-ui-design-lead in design system extraction mode on this existing app and identify reusable components, token categories, and the highest-value normalization work.`
- `Use $ux-ui-design-lead in accessibility pass mode on this running web app and focus on keyboard flow, focus states, and form semantics.`
- `Use $ux-ui-design-lead in implementation guidance mode on this codebase and produce a DESIGN.md handoff for the frontend agent.`
- `Use $ux-ui-design-lead in polish mode on these desktop screenshots and identify where spacing, alignment, and emphasis make the product feel unfinished.`
- `Use $ux-ui-design-lead in component review mode on this table and filter bar and tell me what the current design communicates versus what it should communicate.`
- `Use $ux-ui-design-lead with the frontend_design_reviewer specialist to inspect the running app, compare the DOM and visible hierarchy, and recommend direct UI fixes.`
- `Use $ux-ui-design-lead with the flow_simplifier specialist to shorten this multi-step setup flow without hiding critical decisions.`

## Evidence Packaging Examples

### Screenshot-Only

`Use $ux-ui-design-lead on these screenshots. Infer the primary user job, call out hierarchy and semantic mismatches, and propose a simpler target structure.`

### Running App

`Use $ux-ui-design-lead in audit mode on the running app at this route. Inspect the browser state, compare intended hierarchy versus actual hierarchy, and return the full review contract.`

### Code-Only

`Use $ux-ui-design-lead in implementation guidance mode from this codebase alone. Infer likely user intent carefully, label assumptions, and produce a DESIGN.md handoff.`

### Bug Report

`Use $ux-ui-design-lead on this bug report about confusing permissions settings. Explain the semantic failure and propose the smallest safe redesign.`
