# Feedback Loops

## Goal

Measure whether Codex is actually creating leverage.

## Minimum Loop

For each initiative:

1. define the hypothesis
2. define the success metrics
3. set a review window
4. compare expected versus actual effect
5. keep, refine, or kill the initiative

## Useful Metrics

- reduction in missed follow-ups
- reduction in time spent reconstructing context
- faster time from signal to action
- fewer user corrections
- more decisions closed
- more useful drafts or briefs produced before being asked
- measurable improvement in technical bottlenecks under Codex ownership

## User Feedback

The system may periodically ask:

- was this initiative useful
- what was too noisy
- what should become more autonomous
- what should be scaled back

Keep feedback requests short and tied to specific initiatives.

## Kill Criteria

Kill or pause an initiative when:

- the user ignores it repeatedly
- it creates more review burden than leverage
- the signal is too noisy
- the source data is too stale or weak
- the domain no longer matters
