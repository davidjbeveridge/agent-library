# Initiative Charters

## Purpose

Standing charters let Codex act within a bounded domain without requiring approval for every tool call.

## Charter Types

### Observation Charter

Allows Codex to:

- read approved sources
- refresh context
- maintain a live operating picture
- surface changes that matter

### Recommendation Charter

Allows Codex to:

- rank opportunities
- draft briefs, memos, plans, and options
- recommend initiatives or follow-ups

### Execution Charter

Allows Codex to:

- perform bounded actions inside an approved initiative
- use allowed tools and systems
- complete routine follow-through without repeated approval

## Required Fields

- objective
- domain
- scope
- allowed systems
- allowed actions
- escalation-only actions
- reporting cadence
- stop conditions
- success metrics

## Good Charter Pattern

Good:

- “Own weekly operating hygiene for my leadership work across Slack, calendar, docs, and durable memory. You may prepare briefs, draft follow-ups, create low-risk reminders, and maintain task state. Escalate before sending sensitive people-management messages or changing external commitments.”

Bad:

- “Just do whatever seems useful.”

## Escalation Examples

Always escalate if the charter would otherwise permit:

- sensitive personnel actions
- strategic commitments on behalf of the user
- spending money
- external publication
- broad scope expansion

## Working Rule

Broad authority is acceptable only when the domain, boundaries, and reporting are explicit.
