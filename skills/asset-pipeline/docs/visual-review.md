# Visual Review

Visual QA in this bundle is semantic first, technical second.

## Review Dimensions

### Semantic Communication

- does the asset communicate the intended action, concept, or message
- is the icon or mark understandable without insider knowledge
- does the diagram or chart express the dominant relationship clearly

### Hierarchy And Readability

- is the focal point obvious
- is text readable at intended size
- is contrast adequate for the target context

### Brand Consistency

- does the asset fit the stated brand tone
- do stroke, corner, spacing, and color decisions align with adjacent assets

### Technical Quality

- are edges clean
- are dimensions and aspect ratios correct
- are exports crisp at intended sizes
- is the SVG or text spec maintainable

### Optimization Quality

- was file size reduced without meaningful quality loss
- was metadata stripped only when safe
- are derivatives named and organized consistently

## Output Style

Review reports should include:

- what the asset appears to communicate
- what it fails to communicate
- technical defects
- concrete next edits
- confidence level when context is incomplete
