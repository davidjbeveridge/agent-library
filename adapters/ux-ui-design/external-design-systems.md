# Optional External Design Systems Adapter

This adapter is optional. The core pack does not require any external design system.

## Use When

- an existing external design system must be interpreted for local implementation
- the team needs to map outside component patterns into the pack's design language

## Inputs

- external component guidance
- local product requirements
- current local UI or code

## Outputs

- translated component rules
- compatibility notes
- risks where external patterns conflict with local user jobs or semantics

## Translation Rules

Map external guidance into the native evidence model:

- user jobs served
- semantic meaning
- hierarchy implications
- interaction expectations
- accessibility expectations
- implementation constraints

Do not inherit external system rules blindly. Translate them into the native evidence model and adapt them to the product's actual task model, accessibility requirements, and implementation constraints.
